# Logging

## Philosophy

- Logs go to stdout. Always. The operating environment (Docker, systemd, Kubernetes,
  bare metal with a log shipper) decides what to do with them.
- JSON in production. Human-readable text in development. Same logger, different handler.
- Every log entry is structured. No free-form concatenated strings.
- Logs must be useful for debugging without ever exposing secrets.

---

## Output

**Stdout only.** No file writing, no rotation. This is a 12-factor app.

Users who want log files use their OS/container tooling:
- **systemd**: `journald` captures stdout automatically → `journalctl -u luminarr`
- **Docker**: `docker logs luminarr` or configure a log driver (`--log-driver=json-file`)
- **Kubernetes**: `kubectl logs` — standard log aggregation picks up stdout
- **Bare metal with Loki**: `promtail` tails the process's stdout
- **Vector / Fluentd / Logstash**: point at the process stdout or Docker socket

## Format

### Production (default): JSON via slog

```json
{"time":"2025-06-01T14:32:01.234Z","level":"INFO","msg":"grab started","movie_id":"550e...","release":"Inception.2010.2160p.BluRay.REMUX.x265-GROUP","indexer":"Prowlarr","size_bytes":58432145000}
{"time":"2025-06-01T14:32:01.891Z","level":"INFO","msg":"outbound request","method":"POST","url":"http://localhost:8080/api/torrents/add","status":200,"duration_ms":142}
{"time":"2025-06-01T14:32:05.100Z","level":"WARN","msg":"download client unreachable","client":"qBittorrent","error":"connection refused"}
```

### Development: text via slog's TextHandler

```
2025-06-01T14:32:01.234Z INFO  grab started movie_id=550e... release=Inception.2010... indexer=Prowlarr
2025-06-01T14:32:01.891Z INFO  outbound request method=POST url=http://localhost:8080/api/torrents/add status=200 duration_ms=142
2025-06-01T14:32:05.100Z WARN  download client unreachable client=qBittorrent error="connection refused"
```

Selected via config:
```yaml
log:
  level: info          # debug | info | warn | error
  format: json         # json | text  (default: json)
```

Or environment variable: `LUMINARR_LOG_FORMAT=text LUMINARR_LOG_LEVEL=debug`

## Log Levels

| Level | Use for |
|-------|---------|
| ERROR | Something failed that requires user attention. Grab failed, import failed, DB error. |
| WARN  | Degraded operation. Download client unreachable, indexer returning errors, disk low. |
| INFO  | Normal significant events. Grab started, import complete, task started/finished, startup. |
| DEBUG | Detailed tracing for troubleshooting. RSS entries evaluated, quality parsing steps, AI call details. |

Default level is `info`. Debug produces significant volume and is for troubleshooting only.

## Standard Fields

Every log entry from application code should include relevant structured fields.
Never concatenate values into the message string.

```go
// Good
slog.Info("grab started",
    "movie_id", movie.ID,
    "release", release.Title,
    "indexer", release.Indexer,
    "size_bytes", release.Size,
)

// Bad — unstructured, unsearchable
slog.Info(fmt.Sprintf("grabbing %s from %s", release.Title, release.Indexer))
```

### Common Field Names (use these consistently)

```
movie_id        UUID of the movie
library_id      UUID of the library
release         Scene release title
indexer         Indexer name
client          Download client name
task            Task/job name
method          HTTP method
url             HTTP URL (sanitized)
status          HTTP status code
duration_ms     Duration in milliseconds
error           Error message (from err.Error())
path            Filesystem path
size_bytes      File or release size in bytes
```

### Request ID

Every HTTP request gets a UUID request ID generated by middleware. It is:
- Added to the request context
- Included in all log lines emitted during that request's handling
- Returned in the `X-Request-ID` response header

This makes it possible to trace all log lines for a single API call in a log aggregator.

```go
// middleware/logging.go
func RequestID(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        id := uuid.New().String()
        ctx := context.WithValue(r.Context(), ctxKeyRequestID, id)
        w.Header().Set("X-Request-ID", id)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}
```

## What Is Logged

### HTTP Requests (every request)
```
INFO  request method=GET path=/api/v1/movies status=200 duration_ms=12 request_id=abc-123
```
The `X-Api-Key` header value is NEVER logged. Only its presence/absence.

### Outbound HTTP (every call to external service)
```
INFO  outbound request method=GET url=https://api.themoviedb.org/3/movie/27205 status=200 duration_ms=88
```
Auth parameters and headers are stripped before logging. See [11-security-privacy.md](11-security-privacy.md).

### Scheduled Tasks
```
INFO  task started  task=rss_sync
INFO  task finished task=rss_sync duration_ms=1842 grabs=2 errors=0
WARN  task failed   task=rss_sync error="indexer Prowlarr unreachable"
```

### Grab Events
```
INFO  grab started    movie_id=550e... release=Inception.2010.2160p... indexer=Prowlarr quality="2160p BluRay Remux"
INFO  grab sent       movie_id=550e... client=qBittorrent client_item_id=abc123
WARN  grab failed     movie_id=550e... release=Inception... error="torrent already exists"
```

### Import Events
```
INFO  import started  movie_id=550e... source=/downloads/Inception.2010.2160p... size_bytes=58432145000
INFO  import complete movie_id=550e... dest=/mnt/movies/Inception (2010)/Inception.2010.2160p.BluRay.REMUX.mkv method=hardlink
WARN  import failed   movie_id=550e... error="insufficient disk space"
```

### AI Calls (DEBUG level)
```
DEBUG ai score requested movie_id=550e... release=Inception.2010.2160p... model=claude-haiku-4-5
DEBUG ai score result    movie_id=550e... score=87 input_tokens=312 output_tokens=48 duration_ms=210
```
Full prompts and responses are NOT logged.

### Startup
```
INFO  Luminarr starting version=0.1.0 go=go1.24.0 db=sqlite
INFO  database migrations complete version=5
INFO  registered indexer plugin plugin=torznab
INFO  registered indexer plugin plugin=newznab
INFO  registered downloader plugin plugin=qbittorrent
INFO  registered downloader plugin plugin=transmission
INFO  registered notifier plugin plugin=discord
INFO  AI features enabled model=claude-haiku-4-5
WARN  TMDB API key not configured — metadata features disabled
INFO  HTTP server listening addr=:7878
```

## What Is Never Logged

See [11-security-privacy.md](11-security-privacy.md) for the complete list.

Short version: no secrets, no auth header values, no raw plugin settings JSON,
no full HTTP bodies, no full AI prompts or responses.

## Piping to Log Aggregators

Since logs are JSON to stdout, no Luminarr-specific configuration is needed.

### Grafana Loki + Promtail (bare metal)

```yaml
# promtail config snippet
scrape_configs:
  - job_name: luminarr
    static_configs:
      - targets: [localhost]
        labels:
          job: luminarr
          __path__: /proc/$(pgrep luminarr)/fd/1  # stdout via /proc
    pipeline_stages:
      - json:
          expressions:
            level: level
            msg: msg
```

Or simpler: run Luminarr under systemd, promtail tails the journal.

### Docker + Loki

```yaml
# docker-compose.yml
services:
  luminarr:
    image: luminarr:latest
    logging:
      driver: loki
      options:
        loki-url: "http://localhost:3100/loki/api/v1/push"
```

### Vector (universal log shipper)

```toml
# vector.toml
[sources.luminarr]
type = "docker_logs"
include_containers = ["luminarr"]

[transforms.parse]
type = "remap"
inputs = ["luminarr"]
source = '''
. = parse_json!(.message)
'''

[sinks.datadog]
type = "datadog_logs"
inputs = ["parse"]
```

## slog Setup

```go
// internal/logging/logging.go

func New(cfg Config) *slog.Logger {
    level := parseLevel(cfg.Level)          // info by default

    var handler slog.Handler
    opts := &slog.HandlerOptions{Level: level}

    switch cfg.Format {
    case "text":
        handler = slog.NewTextHandler(os.Stdout, opts)
    default: // "json"
        handler = slog.NewJSONHandler(os.Stdout, opts)
    }

    return slog.New(handler)
}
```

The logger is constructed in `main.go` and passed to services that need it. It is
also set as the default logger (`slog.SetDefault`) so that packages using the global
`slog` functions pick up the configured handler.
